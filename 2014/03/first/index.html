我建立的第一个网站

我建立的一个博客，记录我学习历程.

first

Mon Mar 10, 2014 by 尹向旭

在personal-web的命令提示符中依次输入：

  • hugo server –watch –theme=personal-web
  • myblog »hugo server -t personal-web –buildDrafts
  • myblog »hugo –theme=personal-web –baseUrl="https://yinxx.github.io” –buildDrafts

public根目录下依次输入：

  • git init 将public文件夹转换为仓库

  • git add . 选择所有的内容

  • git commit -m “first commit” 提交至个人站点中

  • 第一条：git init

  • 第二条：git add . （切记：add后面加一个空格再加一个“.”）

  • 第三条：git commit -m “我的hugo博客”

  • 将本地public内容与站点进行关联,后面的地址是:GitHub官网+你的账户名+你新建的站点名+.git

  • 第四条：git remote add origin https://github.com/yinxx/yinxx.github.io.git

  • 将public刷新到站点中 地址与上面相同.第一次会叫你输入你GitHub的账户密码.密码是暗文照常输入就行

  • 第五条：git push -u origin master

  • 接下来可能会让你输入你的github用户名和密码，在弹出的提示符后输入自己github的用户名和密码就大功告成了，可以去github看是否成功。还可以通过yinxx.github.io来直接访问你搭建好的博客（网址即是新建的仓库名） ！！！

  • 至于喜欢哪种格式，可以在config.toml中进行配置，默认使用的是yaml格式。通过执行hugo new 命令生成的markdown文件，头部默认会有这段渲染之后的Front Matter，一般我们会把draft属性去掉，draft草稿的意思，有这个属性的md文件不会渲染输出， 当然通过hugo –buildDrafts可以强制输出。

content

存放网页内容的目录，即我们编写的markdown文件都存放在此目录，此目录是Hugo的默认源目录，在mybolg下执行命令 hugo new post/hugo_introduce.md之后， 则会在content目录下生成子目录post，post中有一个hugo_introduce.md文件。

data

data目录用来存放数据文件，一般是json文件，Hugo提供了相关命令可以从data目录下读取相关的文件数据，然后渲染到HTML页面中，将业务数据与模板分离。

layouts

存放自定义的模板文件，Hugo优先使用layouts目录下的模板，未发现再去themes目录下查找。

static

存放静态文件，比如css、js、img等文件目录，Hugo在渲染时，会直接将static目录下的文件直接复制到public目录下，不会做任何渲染。

themes

存放网站主题，可以下载多个主题，themes目录下的每个子目录代表了一个主题，可以通过在config.toml中通过参数theme指定主题，即theme目录下的子目录名字， 也可以在执行hugo命令渲染时通过增加flag参数–theme=xx指定。

  • archetypes ：存放default.md，头文件格式
  • content ：content目录存放博客文章（markdown文件）
  • data ：存放自定义模版，导入的toml文件（或json，yaml）
  • layouts ：layouts目录存放的是网站的模板文件
  • static ：static目录存放图片，css等静态资源
  • themes ：存放网站主题文件
  • config.toml ：config.toml是网站的配置文件
